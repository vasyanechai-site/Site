# Автоматическая проверка статусов оплаты

## 📋 Обзор

Реализована комплексная система автоматической проверки статусов оплаты для розничных заказов с использованием API Точка Банка.

## 🎯 Основные компоненты

### 1. Server Endpoints (`/supabase/functions/server/index.tsx`)

#### `POST /retail/check-payment-status/:orderId`
Проверяет статус конкретного заказа и обновляет его в базе данных.
- Получает статус из Точка Банк API
- Обновляет `paymentStatus` и `status` заказа
- Автоматически начисляет/списывает баллы лояльности
- Возвращает информацию об обновлении

#### `POST /retail/check-pending-payments`
Массовая проверка всех незавершенных заказов.
- Находит все заказы со статусом `pending`
- Проверяет каждый через Точка Банк API
- Обновляет статусы и обрабатывает баллы
- Возвращает статистику обновлений

#### `GET /retail/order-payment-info/:orderId`
Получает полную информацию о платеже для пользователя.
- Статус заказа и оплаты
- Начисленные/списанные баллы лояльности
- Текущий баланс пользователя

### 2. Cron Function (`/supabase/functions/payment-status-checker/index.ts`)

Автоматическая проверка по расписанию через Supabase Cron.

**Настройка в Supabase Dashboard:**
1. Перейти в Edge Functions → payment-status-checker
2. Settings → Cron
3. Установить расписание: `*/5 * * * *` (каждые 5 минут)
4. Сохранить настройки

- ✅ Показывает актуальные статусы заказов

#### RetailOrderSuccess (`/components/RetailOrderSuccess.tsx`)
- ✅ Автоматическая проверка статуса при переходе на страницу

- ✅ Отображает текущий баланс
- ✅ Адаптивные сообщения в зависимости от статуса

## 🔄 Работа системы

### 1. При создании заказа
```
Заказ создан → Статус: pending → tochkaRequestId сохранен
```

### 2. Автоматическая проверка (3 способа)

#### A. Пользователь открывает dashboard
```
1. Загрузка заказов пользователя
2. Фильтрация pending заказов с tochkaRequestId
3. Для каждого: проверка через Точка API
4. Обновление статусов и баллов
5. Отображение актуальной информации
```

#### B. Администратор открывает админку заказов
```
1. Загрузка всех заказов
2. Автоматический вызов check-pending-payments
3. Toast с результатами проверки
4. Обновление списка заказов
```

#### C. Cron каждые 5 минут
```
1. Supabase Edge Function срабатывает
2. Вызов check-pending-payments endpoint
3. Проверка и обновление всех pending заказов
4. Логирование результатов
```

### 3. Страница успешной оплаты
```
1. Пользователь попадает на /retail/order-success?order_id=...
2. Загрузка информации о заказе
3. Если статус pending - автоматическая проверка
4. Отображение:
   - Статуса оплаты
   - Списанных вушей
   - Начисленных вушей
   - Текущего баланса
```

## 💎 Обработка баллов лояльности

### При подтверждении оплаты:

#### 1. Начисление заработанных баллов
```javascript
if (order.loyaltyPointsEarned > 0 && !order.loyaltyPointsAwarded) {
  currentBalance += order.loyaltyPointsEarned;
  order.loyaltyPointsAwarded = true;
}
```

#### 2. Списание использованных баллов
```javascript
if (order.loyaltyPointsUsed > 0 && !order.loyaltyPointsDeducted) {
  currentBalance -= order.loyaltyPointsUsed;
  order.loyaltyPointsDeducted = true;
}
```

### Защита от дублирования:
- Флаги `loyaltyPointsAwarded` и `loyaltyPointsDeducted`
- Проверки при каждом обновлении
- Идемпотентность операций

## 📊 Статусы платежей

### От Точка Банк API:
- `paid` / `success` / `completed` / `authorized` → **paid**
- `failed` / `error` → **failed**
- `cancelled` / `canceled` → **cancelled**
- Все остальные → **pending**

### В системе:
- `paymentStatus`: состояние платежа
- `status`: общий статус заказа
- После оплаты: `paymentStatus = paid`, `status = processing`

## 🚀 Преимущества системы

1. **Автоматизация**: Статусы обновляются без ручного вмешательства
2. **Надежность**: Три независимых способа проверки
3. **Прозрачность**: Пользователь всегда видит актуальную информацию
4. **Безопасность**: Защита от двойного начисления/списания баллов
5. **UX**: Мгновенная обратная связь после оплаты

### Troubleshooting

**Проблема**: Статус не обновляется
- Проверьте наличие `tochkaRequestId` в заказе
- Проверьте доступность API Точка Банка
- Проверьте корректность JWT токена

**Проблема**: Баллы начислены дважды
- Проверьте флаги `loyaltyPointsAwarded` и `loyaltyPointsDeducted`
- Посмотрите логи обработки заказа

**Проблема**: Cron не работает
- Убедитесь что настроен в Supabase Dashboard
- Проверьте логи Edge Function
- Проверьте правильность расписания
