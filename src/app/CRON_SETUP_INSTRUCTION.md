# 🕐 Настройка Cron для автоматической проверки платежей

## Быстрая настройка (5 минут)

### Шаг 1: Откройте Supabase Dashboard
1. Перейдите на https://supabase.com
2. Откройте ваш проект
3. В боковом меню выберите **Edge Functions**

### Шаг 2: Разверните Edge Function
1. В списке функций найдите **payment-status-checker**
2. Если функции нет:
   ```bash
   supabase functions deploy payment-status-checker
   ```

### Шаг 3: Настройте Cron
1. Нажмите на функцию **payment-status-checker**
2. Перейдите на вкладку **Settings** или **Cron**
3. Включите **Enable Cron**
4. Установите расписание: **`*/5 * * * *`**
   - Это означает: каждые 5 минут
5. Нажмите **Save** или **Update**

### Шаг 4: Проверьте работу
1. Перейдите на вкладку **Logs**
2. Дождитесь следующего запуска (максимум 5 минут)
3. Вы должны увидеть логи с результатами проверки

## Альтернативные расписания Cron

Если хотите изменить частоту проверки:

| Расписание | Описание |
|------------|----------|
| `*/5 * * * *` | Каждые 5 минут (рекомендуется) |
| `*/10 * * * *` | Каждые 10 минут |
| `*/15 * * * *` | Каждые 15 минут |
| `0 * * * *` | Каждый час |
| `0 */2 * * *` | Каждые 2 часа |
| `0 9-18 * * *` | Каждый час с 9:00 до 18:00 |

## Что произойдет после настройки?

✅ Каждые 5 минут система автоматически:
1. Найдет все неоплаченные заказы
2. Проверит их статус в Точка Банке
3. Обновит статусы оплаты
4. Начислит/спишет баллы лояльности
5. Запишет результаты в логи

## Проверка работы Cron

### В Supabase Dashboard:
1. Edge Functions → payment-status-checker → Logs
2. Должны появляться записи каждые 5 минут

### В логах должно быть:
```json
{
  "success": true,
  "timestamp": "2025-01-20T12:00:00.000Z",
  "checked": 5,
  "updated": 2,
  "paid": 2,
  "failed": 0,
  "cancelled": 0
}
```

## Troubleshooting

### Проблема: Cron не запускается
**Решение:**
1. Убедитесь что функция развернута
2. Проверьте что Cron включен в настройках
3. Проверьте правильность синтаксиса расписания

### Проблема: Функция падает с ошибкой
**Решение:**
1. Проверьте логи на наличие ошибок
2. Убедитесь что `TOCHKA_JWT_TOKEN` настроен
3. Проверьте доступность API Точка Банка

### Проблема: Статусы не обновляются
**Решение:**
1. Проверьте что у заказов есть `tochkaRequestId`
2. Проверьте что Точка API возвращает корректный статус
3. Посмотрите детальные логи в Edge Function

## 🎉 Готово!

Пользователи увидят обновленные статусы при следующем входе в личный кабинет!
