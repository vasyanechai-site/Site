Сделай в текущем проекте новый полноценный модуль «Агенты» для бизнеса по обжарке кофе. Бизнес уже работает: есть интернет-магазин, оптовый кабинет, админка с заказами, оплатами, статусами, интеграциями с CDEK и Точкой. Ничего не ломай в текущем проекте, не меняй существующую архитектуру без необходимости, не удаляй текущие компоненты и не переделывай рабочие части сайта. Нужно аккуратно расширить существующую систему.

Задача: реализовать агентский кабинет для партнеров, которые могут рекомендовать мой кофе кофейням, ресторанам и другим бизнесам которым может понадобится мой кофе. Агентом может быть бариста, владелец кофейни или любой человек, который приводит мне новых B2B-клиентов. За заказы клиентов, которых привел агент, агенту начисляется комиссия в процентах от заказа клиента которого он привел. Размер процента я задаю вручную в админке для каждого агента отдельно.

Основная логика:
1. Я как администратор создаю агента в админке.
2. У агента есть имя, логин, пароль, статус и процент комиссии.
3. Агент может авторизоваться через кнопку «Войти для бизнеса» и попасть в свой личный кабинет.
4. В своем ЛК агент может создать нового оптового клиента.
5. У оптового клиента есть название компании, логин, пароль и привязка к агенту, который его создал. - это ровно те же пользователи что сейчас я сам создаю и они в той же таблице должны появляться в админке но со статусом от агента
6. Все заказы, оформленные под этим оптовым клиентом, автоматически считаются заказами, приведенными данным агентом.
7. Система должна считать общую выручку по клиентам агента и сумму комиссии, которую я должен агенту.
8. Я как админ вижу статистику по каждому агенту, могу вручную отметить выплату комиссии, указать сумму выплаты, поменять статус на «оплачено», после чего задолженность агенту должна уменьшаться или обнуляться в зависимости от выплаченной суммы.

Нужно реализовать следующие сущности данных.

Сущность «Агент»:
- id
- имя
- логин
- пароль
- процент комиссии
- статус: active / disabled
- дата создания

Сущность «Оптовый клиент»:
- id
- название компании
- логин
- пароль
- agent_id
- дата создания

Сущность «Выплата агенту»:
- id
- agent_id
- сумма
- статус: pending / paid
- дата создания
- дата выплаты
- комментарий / примечание

Использовать существующую сущность «Заказ», но связать ее с оптовым клиентом. Для каждого заказа должны быть доступны:
- id
- wholesale_client_id
- сумма заказа
- статус
- дата создания

Правила расчета комиссии:
- если заказ оформлен на оптового клиента, у которого есть agent_id, значит заказ относится к агенту
- комиссия по заказу = сумма заказа * процент комиссии агента
- в агентском кабинете и в админке отображать накопленную сумму комиссии
- отдельным полем показывать уже выплаченные суммы
- отдельным полем показывать текущую задолженность перед агентом
- после ручной выплаты задолженность должна корректно пересчитываться
