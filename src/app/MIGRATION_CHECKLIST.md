## 📋 Что нужно сделать ВРУЧНУЮ:

### 1. **Настройки Supabase (ОБЯЗАТЕЛЬНО!)**

Зайдите в: [Supabase Dashboard](https://supabase.com/dashboard) → Ваш проект → Authentication → URL Configuration

Измените:
- **Site URL:** `https://coffeenechai.ru`
- **Redirect URLs:** Добавьте `https://coffeenechai.ru/**` в список

> ⚠️ Без этого не будет работать регистрация и вход розничных пользователей!

### 2. **DNS и домен**

Настройте DNS для coffeenechai.ru, чтобы он указывал на ваш хостинг/CDN, где развернуто приложение.

### 3. **Email адреса (опционально)**

Если хотите использовать новые email адреса:
- Создайте `no-reply@coffeenechai.ru`
- Создайте `support@coffeenechai.ru`
- Настройте почтовый сервер или пересылку

### 4. **SSL сертификат**

Убедитесь, что для coffeenechai.ru настроен HTTPS (Let's Encrypt или другой)

## ⚠️ Важные замечания:

### Что НЕ изменялось (работает автоматически):
- ✅ СДЭК API - не зависит от домена
- ✅ Telegram webhook - формируется из SUPABASE_URL
- ✅ Все API вызовы к Supabase
- ✅ Внешние ресурсы (Tilda CDN, изображения)

### Переменные окружения:
Проверьте, что все переменные окружения Supabase настроены:
- `SUPABASE_URL` - должна остаться прежней (это URL самого Supabase)
- `SUPABASE_ANON_KEY` - не меняется
- `SUPABASE_SERVICE_ROLE_KEY` - не меняется
- `TELEGRAM_BOT_TOKEN` - не меняется
- `TELEGRAM_CHAT_ID` - не меняется
- `TOCHKA_*` - все ключи Tochka не меняются

> 💡 **Важно:** SUPABASE_URL — это адрес вашего Supabase проекта (например, `https://pkhinqiplfezrzvsqgwo.supabase.co`), а НЕ адрес вашего сайта. Он остается без изменений.

## 📞 Что делать, если что-то пошло не так:

1. **Оплата не работает** → Проверьте redirectUrl в консоли браузера
2. **Не работает вход** → Проверьте Site URL в Supabase
3. **Telegram не отвечает** → Проверьте TELEGRAM_BOT_TOKEN в переменных окружения
4. **СДЭК не работает** → Проверьте консоль браузера на ошибки API
