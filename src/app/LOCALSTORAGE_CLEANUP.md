# 🧹 Очистка битых данных в localStorage

## Проблема

Вы видите предупреждение в консоли:
```
⚠️  LocalStorage: найдены битые символы в ключах:
  - nechai_coffee_items
```

Это означает, что в localStorage есть данные с некорректной кодировкой (символы замены `�`).

## ✅ Автоматическое исправление

### Система автоматической очистки

Приложение **автоматически** обнаруживает и очищает битые данные при каждой загрузке страницы:

1. 🔍 Сканирует все ключи localStorage
2. ✅ Находит данные с битыми символами (`\uFFFD`)
3. 🧹 Автоматически удаляет повреждённые данные
4. 📦 Для важных ключей устанавливает значения по умолчанию
5. ✨ Данные будут загружены заново с сервера

### Что происходит автоматически

При обнаружении битых данных:

```
🔧 Автоматическая очистка битых данных...
  ✅ Очищен: nechai_coffee_items
  📦 Установлен пустой массив для nechai_coffee_items
🔄 Обновите страницу для загрузки свежих данных с сервера
```

**Просто обновите страницу (F5)** и данные загрузятся заново с сервера.

## 📋 Какие данные очищаются

Система очищает данные, если:

1. **Битые символы:** содержат Unicode replacement character (`\uFFFD`)
2. **Невалидный JSON:** не могут быть распарсены
3. **Неправильная структура:** не соответствуют ожидаемому формату

## 🔐 Какие данные защищены

При очистке для критически важных ключей устанавливаются значения по умолчанию:

| Ключ | Значение по умолчанию | Описание |
|------|----------------------|----------|
| `nechai_coffee_items` | `[]` | Пустой массив, данные загрузятся с сервера |
| `nechai_orders` | `[]` | Пустой массив, данные загрузятся с сервера |
| `nechai_retail_orders` | `[]` | Пустой массив, данные загрузятся с сервера |

## ⚙️ Валидация при сохранении

Новые данные проверяются **перед** сохранением в localStorage:

```typescript
// В lib/api.ts

const saveLocalCoffeeItems = (items: CoffeeItem[]): void => {
  // ✅ Проверка: массив ли это
  if (!Array.isArray(items)) return;
  
  const jsonString = JSON.stringify(items);
  
  // ✅ Проверка: нет ли битых символов
  if (jsonString.includes('\uFFFD')) return;
  
  // Только валидные данные попадут в localStorage
  localStorage.setItem(COFFEE_ITEMS_KEY, jsonString);
};
```

## 🐛 Откуда берутся битые символы?

Возможные причины:

1. **Неправильная кодировка при сохранении** - данные были сохранены не в UTF-8
2. **Проблемы с API** - сервер вернул данные в неправильной кодировке
3. **Копирование через буфер обмена** - при вставке текста с неподдерживаемыми символами
4. **Некорректные символы в именах** - спецсимволы, которые не поддерживаются
