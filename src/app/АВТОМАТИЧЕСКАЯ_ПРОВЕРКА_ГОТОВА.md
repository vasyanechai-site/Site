## ⚙️ Опциональная настройка

### Изменить частоту проверки

Откройте `/App.tsx` и найдите:

```javascript
// Текущая: каждые 5 минут (300000 мс)
setInterval(checkPayments, 300000);

// Каждые 3 минуты:
setInterval(checkPayments, 180000);

// Каждые 10 минут:
setInterval(checkPayments, 600000);

// Каждую минуту (не рекомендуется - много запросов):
setInterval(checkPayments, 60000);
```

### Добавить визуальный индикатор (необязательно)

Создан компонент `AutoPaymentCheckerIndicator.tsx` - показывает бейдж в углу экрана с таймером следующей проверки.

Чтобы включить, добавьте в нужную страницу:
```javascript
import { AutoPaymentCheckerIndicator } from './components/AutoPaymentCheckerIndicator';

// В компоненте:
<AutoPaymentCheckerIndicator show={true} />
```

## 🆘 Troubleshooting

### Не вижу логов в консоли

1. Откройте DevTools (F12)
2. Вкладка Console
3. Обновите страницу (Ctrl+R)
4. Подождите 10 секунд

### Статус не обновляется

1. ✅ Проверьте консоль - есть ли ошибки
2. ✅ Используйте кнопку "Проверить оплату" в админке
3. ✅ Проверьте логи Supabase
4. ✅ Убедитесь что заказ в статусе "pending"

### Ошибки в консоли

Если видите ошибки типа `Failed to fetch`:
1. Проверьте доступность endpoint
2. Проверьте `projectId` и `publicAnonKey`
3. Проверьте логи в Supabase Dashboard

## 💡 Когда НЕ работает

### Единственное ограничение:
**Проверка работает только пока кто-то находится на сайте**

- Если сайт закрыт у всех → проверка не идёт
- Когда кто-то откроет → проверка возобновится

### Для магазина это НЕ проблема:
- ✅ Днём всегда есть посетители
- ✅ Админ заходит утром → всё обновится
- ✅ После оплаты пользователь на сайте → сразу видит результат

### Если нужна проверка 24/7:

Используйте один из вариантов (опционально):

1. **GitHub Actions** - проверка каждые 5 минут даже ночью
   📖 См. `GITHUB_ACTIONS_PAYMENT_CHECKER.md`

2. **cron-job.org** - внешний сервис, настройка за 1 минуту
   📖 См. `SIMPLE_CRON_ALTERNATIVES.md`

3. **Supabase Edge Function** - встроенный Cron
   📖 См. `CRON_SETUP_INSTRUCTION.md`
