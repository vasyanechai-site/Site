# 🚀 Быстрый старт: Telegram-рассылка

## ⚡ Автоматическая настройка (2 минуты)

### Шаг 1: Создайте таблицу в базе данных

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Перейдите в **SQL Editor**
3. Выполните SQL-запрос:

```sql
CREATE TABLE telegram_users (
  id BIGINT PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

### Шаг 2: Настройте webhook через админ-панель

1. Откройте **админ-панель** вашего магазина
2. Перейдите на вкладку **"Рассылка"** (Broadcast)
3. В блоке **"Настройка Webhook"** нажмите **"Установить webhook"**
4. Дождитесь сообщения об успешной настройке ✅

### Шаг 3: Проверьте работу

1. Напишите любое сообщение вашему боту в Telegram
2. Бот должен ответить: "✅ Вы подписались на рассылку!"
3. В админ-панели (раздел "Рассылка") введите ваш Chat ID и нажмите **"Отправить тестовое сообщение"**

**🎉 Готово!** Система полностью настроена и готова к использованию.

## 🔧 Как использовать систему

### 1. Управление webhook

В админ-панели (раздел "Рассылка") доступны функции:

- **Просмотр статуса** webhook (активен/не активен)
- **Установка/переустановка** webhook
- **Удаление** webhook
- **Проверка ошибок** (если есть)
- **Тестирование** отправки сообщений

### 2. Отправка рассылки

В том же разделе ниже:

1. Напишите текст сообщения
2. (Опционально) Добавьте изображение
3. Нажмите **"Отправить рассылку"**

Рассылка будет отправлена **всем пользователям**, которые написали боту хотя бы одно сообщение.

### 3. Управление подписчиками

- Пользователи подписываются автоматически, написав боту
- Список подписчиков хранится в таблице `telegram_users`
- Можно просмотреть подписчиков через Supabase Dashboard → Table Editor

## 🔍 Отладка

### Проверка статуса webhook

**Через админ-панель:**
- Откройте раздел "Рассылка"
- Нажмите кнопку обновления (🔄)
- Проверьте статус и наличие ошибок

**Через Telegram API:**
```bash
curl "https://api.telegram.org/bot<ВАШ_BOT_TOKEN>/getWebhookInfo"
```

### Просмотр логов

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Перейдите в **Functions → telegram-webhook → Logs**
3. Проверьте логи обработки сообщений

### Типичные проблемы

| Проблема | Решение |
|----------|---------|
| Бот не отвечает | Проверьте, что webhook установлен (статус в админ-панели) |
| Ошибка "Table doesn't exist" | Создайте таблицу `telegram_users` (см. Шаг 1) |
| Рассылка не отправляется | Убедитесь, что есть подписчики (напишите боту сообщение) |
| Ошибка 500 в webhook | Проверьте логи Edge Function в Supabase |

```
┌─────────────┐
│  Telegram   │
│    User     │
└──────┬──────┘
       │ Отправляет сообщение боту
       ↓
┌─────────────────────────────────┐
│   Telegram API (Webhook)        │
└──────────┬──────────────────────┘
           │
           ↓
┌─────────────────────────────────┐
│  Edge Function                  │
│  /telegram-webhook              │
│                                 │
│  1. Получает chat_id            │
│  2. Сохраняет в telegram_users  │
│  3. Отвечает пользователю       │
└──────────┬──────────────────────┘
           │
           ↓
┌─────────────────────────────────┐
│  PostgreSQL Database            │
│  Таблица: telegram_users        │
└─────────────────────────────────┘
```

### Переменные окружения

Уже настроены в Supabase:
- `TELEGRAM_BOT_TOKEN` - токен бота (используется для заявок)
- `SUPABASE_URL` - URL проекта Supabase
- `SUPABASE_ANON_KEY` - публичный ключ Supabase

### Endpoints

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/make-server-aa167a09/telegram/webhook/setup` | Установить webhook |
| GET | `/make-server-aa167a09/telegram/webhook/info` | Получить информацию о webhook |
| DELETE | `/make-server-aa167a09/telegram/webhook/delete` | Удалить webhook |
| POST | `/make-server-aa167a09/telegram/webhook/test` | Тестировать отправку |
| POST | `/make-server-aa167a09/telegram/broadcast` | Отправить рассылку |
