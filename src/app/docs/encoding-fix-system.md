# Система мониторинга и исправления битых текстов

## Описание проблемы

В проекте иногда появлялись битые тексты из-за проблем с кодировкой UTF-8. Например:
- `аша корзина пуста` вместо `Ваша корзина пуста`
- `азы` вместо `Заказы`
- `бязательство` вместо `Обязательство`

## Решение

Реализована автоматическая система мониторинга и исправления битых текстов на всех страницах и в модальных окнах.

### Компоненты системы

#### 1. EncodingValidator (`/components/EncodingValidator.tsx`)
Основной компонент для автоматического мониторинга DOM и исправления битых текстов в реальном времени.

**Возможности:**
- Сканирование всех текстовых узлов на странице при загрузке
- Отслеживание изменений DOM через MutationObserver
- Автоматическое исправление битых текстов
- Логирование обнаруженных проблем в консоль (в dev режиме)

**Как работает:**
1. При монтировании компонент сканирует все текстовые узлы в document.body
2. Проверяет наличие replacement character `` (U+FFFD)
3. Применяет известные замены из карты `ENCODING_FIXES`
4. Автоматически исправляет найденные битые тексты
5. MutationObserver отслеживает новые добавленные узлы и сканирует их

**Использование:**
```tsx
import { EncodingValidator } from './components/EncodingValidator';

function App() {
  return (
    <ThemeProvider>
      <EncodingValidator /> {/* Добавить в корень приложения */}
      <BrowserRouter>
        {/* остальной код */}
      </BrowserRouter>
    </ThemeProvider>
  );
}
```

#### 2. Утилиты исправления (`/scripts/fix-encoding.ts`)
Вспомогательные функции для ручного использования.

**Функции:**
- `fixText(text: string)` - исправляет битый текст
- `hasBrokenEncoding(text: string)` - проверяет наличие битых символов
- `fixObjectEncoding<T>(obj: T)` - рекурсивно исправляет все строки в объекте
- `useFixedText(text: string)` - React hook для автоматического исправления

**Пример использования:**
```tsx
import { fixText, useFixedText } from '../scripts/fix-encoding';

// Ручное исправление
const title = fixText('аша корзина');
console.log(title); // "Ваша корзина"

// В React компоненте
function MyComponent({ text }: { text: string }) {
  const fixed = useFixedText(text);
  return <h1>{fixed}</h1>;
}
```

### Карта замен (ENCODING_FIXES)

Содержит известные битые последовательности и их исправления:

```typescript
const ENCODING_FIXES: Record<string, string> = {
  'аша': 'Ваша',
  'аказы': 'Заказы',
  'ромокод': 'Промокод',
  'бязательство': 'Обязательство',
  'ильтр': 'Фильтр',
  // и другие...
};
```

При обнаружении новых битых текстов - добавляйте их в эту карту.

## Мониторинг

### В Development режиме:
- При обнаружении битого текста показывается toast уведомление
- Все исправления логируются в консоль
- Формат лога: `❌ "битый текст" → ✅ "исправленный текст"`

### В Production режиме:
- Toast уведомления отключены (не беспокоят пользователей)
- Логирование в консоль остаётся для отладки

### Когда использовать ручное исправление:
- Для данных, приходящих с сервера

- В утилитах и функциях вне React компонентов


## Предотвращение проблем

### Для разработчиков:
1. Всегда используйте UTF-8 кодировку в редакторе
2. При копировании текстов проверяйте кодировку
3. Избегайте копирования из документов Word/PDF напрямую
4. Используйте валидацию UTF-8 перед коммитом

### VS Code настройки:
```json
{
  "files.encoding": "utf8",
  "files.autoGuessEncoding": false
}
```

## Расширение системы

### Добавление новых замен:
1. Откройте `/components/EncodingValidator.tsx`
2. Добавьте новую запись в `ENCODING_FIXES`:
   ```typescript
   const ENCODING_FIXES: Record<string, string> = {
     'ваш_битый_текст': 'правильный_текст',
     // ...существующие замены
   };
   ```

### Расширение мониторинга:
Компонент уже отслеживает:
- Статические тексты при загрузке страницы
- Динамически добавляемые узлы (модальные окна, попапы)
- Изменения при навигации между страницами

Не требуется дополнительная настройка для новых компонентов.

## Производительность

- Первоначальное сканирование: ~10-50ms (зависит от размера страницы)
- MutationObserver: минимальное влияние (debounce 100ms)
- Память: хранится лог последних 100 исправлений

## Устранение проблем

### Бесконечный цикл исправлений

**Симптомы:** Счетчик исправлений постоянно растет (65, 66, 67...), система непрерывно находит одни и те же битые тексты.

**Причины:**
1. MutationObserver срабатывает на собственные изменения текста
2. Слишком час��ое сканирование без debounce
3. Источник битых текстов в коде постоянно генерирует новые битые символы

**Решение (реализовано):**
1. ✅ Добавлен агрессивный debounce (500ms между сканированиями)
2. ✅ MutationObserver теперь срабатывает только для модальных окон (data-radix-portal)
3. ✅ Сканирование происходит при изменении маршрута, а не постоянно
4. ✅ characterData: false - не отслеживаем изменения текста
5. ✅ Исправлены источники битых текстов в серверных файлах

**Настройки мониторинга:**
```typescript
// Минимальный интервал между сканированиями
const MIN_SCAN_INTERVAL = 500; // мс

// Сканирование происходит только при:
1. Первоначальной загрузке страницы (500ms delay)
2. Изменении маршрута (300ms delay)  
3. Открытии модального окна (300ms debounce)
```
