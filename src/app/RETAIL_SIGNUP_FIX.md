# Исправление регистрации розничных пользователей

## Проблема
Письма подтверждения регистрации не приходили пользователям, так как в Supabase не настроен SMTP сервер для отправки email.

## Решение
Реализован серверный endpoint `/retail-signup`, который использует Supabase Admin API для создания пользователей с автоматическим подтверждением email (параметр `email_confirm: true`).

## Изменения

### 1. Добавлен серверный endpoint (index.tsx)
```typescript
app.post(`${prefix}/retail-signup`, async (c) => {
  // Использует SERVICE_ROLE_KEY для Admin API
  // Создает пользователя с email_confirm: true
  // Возвращает успешный результат без необходимости подтверждения по email
});
```

### 2. Обновлена форма регистрации (SignupForm.tsx)
- Использует серверный endpoint вместо прямого вызова `supabase.auth.signUp`
- Сообщает пользователю, что он может сразу войти после регистрации
- Не показывает сообщение о необходимости проверки почты

### 3. Локализация ошибок
Добавлена обработка и перевод стандартных ошибок Supabase на русский язык:
- "User already registered" → "Пользователь с таким email уже зарегистрирован"
- "Password" → "Пароль должен содержать минимум 6 символов"
- "Email" → "Некорректный email адрес"

## Как это работает

1. Пользователь заполняет форму регистрации (email + пароль)
2. Форма отправляет запрос на сервер `/make-server-aa167a09/retail-signup`
3. Сервер использует Admin API для создания пользователя с подтвержденным email
4. Пользователь получает сообщение об успешной регистрации
5. Пользователь может сразу войти в систему

## Важно
- SMTP сервер **не настроен** в проекте
- Письма **не отправляются** пользователям
- Email подтверждается **автоматически** на сервере
- Это **временное** решение для прототипа
- Для production рекомендуется настроить SMTP сервер в Supabase

## Настройка SMTP (для production)

Для настройки отправки email в Supabase:
1. Перейдите в панель Supabase → Project Settings → Auth
2. Настройте SMTP сервер (рекомендуется Resend, SendGrid или Mailgun)
3. После настройки можно вернуться к стандартному методу регистрации через `supabase.auth.signUp`

## Тестирование
Для тестирования регистрации:
1. Откройте `/signup`
2. Введите email и пароль (минимум 6 символов)
3. Нажмите "Зарегистрироваться"
4. После успешной регистрации сразу можно войти через `/login`
