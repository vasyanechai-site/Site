## 🔐 Уровни защиты

### Уровень 1: Переменные окружения
```
TELEGRAM_BOT_TOKEN  → Хранится в Supabase Secrets
                    → Доступен только серверным функциям
                    → НЕ передаётся на фронтенд
```

### Уровень 2: Серверные эндпоинты
```
/telegram/webhook   → Принимает данные только от Telegram
/broadcast-stats    → Вызывается только из админки
/broadcast          → Используется только админом
```

### Уровень 3: Авторизация админки
```
/admin              → Проверяет localStorage['adminAuth']
                    → Требует валидную сессию (expiresAt)
                    → Редирект на /login если не авторизован
```

### Уровень 4: CORS и заголовки
```
Server CORS         → Разрешает запросы только с домена приложения
Authorization       → Bearer token в заголовке запроса
```

## 📊 Сравнение "До" и "После" аудита

### ДО аудита безопасности ❌
```
Frontend (public) ──┐
                    ├──> api.telegram.org/bot<TOKEN>/...
Telegram Bot Token──┘     ↑
(в клиентском коде)       │
                     Утечка токена и данных!
```

### ПОСЛЕ аудита ✅
```
Frontend (public) ──────────────────────X (нет доступа)
                                        
Frontend (admin) ──> /broadcast-stats ──> Server ──> KV Store
                                           │
                     Telegram Bot Token <──┘
                     (в env переменных)
                                           │
                                           ▼
                                    api.telegram.org
```
