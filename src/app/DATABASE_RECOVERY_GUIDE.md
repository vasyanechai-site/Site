## Шаг 5: Включите RLS (Row Level Security) - ОПЦИОНАЛЬНО

⚠️ Для прототипа это не критично, но для продакшена рекомендуется.

Для таблицы `kv_store_22d84083`:

```sql
-- Включаем RLS
ALTER TABLE kv_store_22d84083 ENABLE ROW LEVEL SECURITY;

-- Создаём политику для чтения (все могут читать)
CREATE POLICY "Allow public read access" 
ON kv_store_22d84083 
FOR SELECT 
USING (true);

-- Создаём политику для записи (только аутентифицированные)
CREATE POLICY "Allow authenticated write access" 
ON kv_store_22d84083 
FOR ALL 
USING (auth.role() = 'service_role' OR auth.role() = 'authenticated');
```

## Часто задаваемые вопросы

### Q: Почему используется одна таблица kv_store вместо отдельных таблиц?
A: Это key-value хранилище для быстрого прототипирования. Все данные хранятся в формате JSON.

### Q: Как сделать бэкап данных?
A: Можно экспортировать таблицу kv_store_22d84083 в CSV через Supabase Dashboard:
   Table Editor → kv_store_22d84083 → Export to CSV

### Q: Можно ли мигрировать на отдельные таблицы?
A: Да, но это потребует переписывания серверного кода и миграции данных.
