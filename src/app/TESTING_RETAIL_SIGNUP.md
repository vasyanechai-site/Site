# Тестирование регистрации розничных пользователей

## Изменения
✅ Регистрация через серверный API с автоматическим подтверждением email  
✅ Не требуется настройка SMTP  
✅ Пользователь может сразу войти после регистрации  
✅ Русские сообщения об ошибках  

## Инструкция для тестирования

### 1. Регистрация нового пользователя

1. Откройте страницу регистрации: `/auth/signup`
2. Введите email (например, `test@example.com`)
3. Введите пароль (минимум 6 символов)
4. Нажмите **"Зарегистрироваться"**

**Ожидаемый результат:**
- Появится сообщение "Регистрация успешна! Теперь вы можете войти"
- Откроется экран с кнопкой "Войти в систему"
- **Письмо НЕ придет** (это нормально - SMTP не настроен)

### 2. Вход после регистрации

1. Нажмите кнопку **"Войти в систему"** на экране успеха
2. Или перейдите на `/auth/login`
3. Введите ваш email и пароль
4. Нажмите **"Войти"**

**Ожидаемый результат:**
- Успешный вход
- Переход на страницу `/dashboard`

### 3. Проверка ошибок

#### Попытка зарегистрироваться с существующим email:
1. Откройте `/auth/signup`
2. Введите email уже зарегистрированного пользователя
3. Нажмите "Зарегистрироваться"

**Ожидаемый результат:**
- Ошибка: "Пользователь с таким email уже зарегистрирован"

#### Слишком короткий пароль:
1. Откройте `/auth/signup`
2. Введите пароль короче 6 символов
3. Нажмите "Зарегистрироваться"

**Ожидаемый результат:**
- Ошибка: "Пароль должен содержать минимум 6 символов"

#### Неверный email или пароль при входе:
1. Откройте `/auth/login`
2. Введите неверные данные
3. Нажмите "Войти"

**Ожидаемый результат:**
- Ошибка: "Неверный email или пароль"

## Проверка в админке Supabase

1. Откройте панель Supabase: https://supabase.com/dashboard
2. Перейдите в **Authentication → Users**
3. Найдите созданного пользователя

**Что проверить:**
- ✅ Email подтвержден (галочка в колонке "Confirmed")
- ✅ Metadata содержит `role: 'retail'`

## Примечания

### ✅ Работает
- Регистрация через серверный API
- Автоматическое подтверждение email
- Вход сразу после регистрации
- Русские сообщения об ошибках

### ⚠️ Не работает (и это нормально)
- Отправка писем подтверждения (SMTP не настроен)
- Сброс пароля через email (требует SMTP)

### 🔧 Для production
Чтобы включить отправку писем:
1. Настройте SMTP в Supabase (Settings → Auth → Email)
2. Измените `SignupForm.tsx` для использования `supabase.auth.signUp`
3. Уберите endpoint `/retail-signup` или оставьте как fallback
