# Как запустить сайт, если вы не программист

Здесь только то, что **вам** нужно сделать руками. Остальное в проекте уже настроено командами.

**Сайт в интернете на домене (бэкенд + фронт + СДЭК + Точка + Telegram):** откройте файл **`ПРОДАКШН-100-ПРОЦЕНТОВ.md`** — там полный чеклист и что уже делает GitHub за вас.

---

## Слова на минуту

- **Терминал** — чёрное или белое окно, куда пишут команды (на Mac: «Терминал», в Windows: «cmd» или PowerShell).
- **Папка проекта** — там лежит сайт. У вас это обычно:  
  `/Users/deniskolosovskii/Sites/Site`
- **Локально** — сайт только на вашем компьютере (для проверки).
- **В интернете** — сайт на хостинге; там нужен доступ к серверу или помощь поддержки хостинга.

---

## Часть 1. Один раз установить Node.js

1. Откройте в браузере: **https://nodejs.org/**
2. Скачайте версию **LTS** (рекомендуемая).
3. Установите как любую программу (Далее — Далее).
4. Проверка: откройте **Терминал** и вставьте:

```bash
node -v
```

Если видите что-то вроде `v22.x` или `v20.x` — всё хорошо.

---

## Часть 2. Один раз подготовить проект на компьютере

1. Откройте **Терминал**.
2. Скопируйте **целиком** эти две строки (путь уже ваш):

```bash
cd /Users/deniskolosovskii/Sites/Site
npm install
```

Подождите, пока закончится (может занять минуту).

3. Если файла **`.env`** ещё не было, выполните:

```bash
npm run env:init
```

Потом по подсказкам в терминале (или у того, кто вам настраивал проект) заполните **`.env`** — это пароли и адреса **только на вашем диске**, в интернет они сами не улетают.

---

## Часть 3. Каждый раз открыть сайт у себя на компьютере

В терминале **две команды**:

```bash
cd /Users/deniskolosovskii/Sites/Site
npm run go
```

Подождите 10–20 секунд, пока в окне не появятся строки про «listening» и Vite.

Откройте браузер и введите адрес **из терминала**, в блоке `[web]`, строка **«Local:»** — например:

- **http://localhost:5173/** — обычный случай  
- **http://localhost:5174/** — если написано `Port 5173 is in use` (порт 5173 уже занят другой программой)

Оба варианта нормальны. При `NODE_ENV=development` в `.env` API принимает запросы с любого локального порта.

Чтобы **освободить 5173** (необязательно): закройте лишние окна с `npm run dev` или в терминале выполните  
`lsof -ti tcp:5173 | xargs kill -9` (осторожно: закроет всё, что слушает этот порт).

Чтобы **остановить** сайт: вернитесь в терминал с `npm run go` и нажмите **Ctrl+C** (на Mac тоже Ctrl+C).

Команда `npm run go` сама:
- проверяет настройки;
- запускает «мозг» сайта (API) и саму страницу (Vite).

## Часть 4. Положить сайт в интернет (коротко)

Тут **нельзя** нажать одну кнопку у себя на Mac и чудом оказаться в проде: нужен **ваш хостинг** (сервер), куда выкладывают файлы и куда смотрит домен.

### Что вы можете сделать сами без «кодинга»

1. В терминале:

```bash
cd /Users/deniskolosovskii/Sites/Site
npm run zip:dist
```

2. В папке проекта появится файл **`site-dist-for-upload.zip`**.
3. Зайдите в **панель хостинга** (Reg.ru, Timeweb, Beget и т.п.) → **файловый менеджер** или **FTP**.
4. Загрузите этот zip в папку сайта (часто `public_html` или `www`) и **распакуйте** архив так, как вам скажет поддержка хостинга, чтобы главная страница открывалась по вашему домену.

#### Важно: «белый экран» после заливки

В репозитории в корне лежит **`index.html` для разработки** — в нём скрипт вида `/src/main.tsx`. Его **нельзя** подменять ими продакшен на обычном хостинге: сервер не отдаст TS, страница останется пустой.

Правило: в интернет попадает только то, что лежит в **`dist/`** после **`npm run build`** (в `dist/index.html` будут пути вида **`/assets/…js`** и **`/assets/…css`**). Команда **`npm run zip:dist`** как раз упаковывает **`dist/`**. Если заливаете вручную через FTP — заливайте **содержимое `dist/`**, не корень проекта.

### Что почти всегда нужно попросить у поддержки хостинга или у администратора

Передайте им **текстом** (можно скопировать из файла в проекте):

- Файл **`deploy/nginx-api-proxy.conf.example`** — там кусок настройки, чтобы адрес **`/api`** на сайте вёл на программу Node.js на сервере.

И отдельно объясните человеку словами:

- «Сайт — это статические файлы из архива; **данные и корзина** идут на **Node** по пути **`/api`**. Нужно, чтобы nginx (или Apache) проксировал `/api` на порт, где запущен Node (часто **8787** или как настроите).»

На сервере должен лежать **тот же по смыслу файл `.env`**, что и у вас локально (секреты, почта, Точка и т.д.), и должна быть команда **перезапуска** этой программы (часто это делает не вы, а тот же администратор).

---

## Часть 5. Что никому не отправлять

- Содержимое файла **`.env`** целиком (там пароли и ключи).
- Строку **`JWT_SECRET`**.
- Пароли от почты и API банка.

Если случайно отправили — сгенерируйте новый секрет: `npm run env:secret` и замените в `.env`.

---

## Шпаргалка команд

| Команда | Зачем |
|--------|--------|
| `npm run go` | Посмотреть сайт на своём Mac |
| `npm run zip:dist` | Собрать сайт и упаковать **архив для загрузки** на хостинг |
| `npm run env:check -- --production` | Проверить, что для «боевого» сервера в `.env` не забыли главное |
| `npm run prod:check` | Как выше **плюс** подсказки по СДЭК, Точке, Telegram, почте, БД |
| `npm run env:secret` | Получить случайный длинный пароль для `JWT_SECRET` |

Если что-то в терминале **красным** напишет «error» — сделайте скриншот и отправьте тому, кто вам помогает с сайтом; или в поддержку Cursor с текстом ошибки.
